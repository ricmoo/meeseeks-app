The meeseeks.app service uses a simple static file, hosted on a CDN designed to allow websites to easily and safely host simple websites from IPFS, optionally using ENS to provide a more human-friendly name, than an IPFS multihash.

What is IPFS?

The Inter-Planetary File System (IPFS) is a large, distributed and decentralized storage service, which allows anyone to freely publish content. Anyone may run a node and contribute to the system, which is where the storage comes from, people around the world dedicating a small portion of their computer's hard drive and bandwidth to the network as a whole.

The content is addressed by the actual contents, so a file with the same contents, has the same "multihash" (think filename). This means if your neighbour recently watched a cat video on YouTube, they likely still have a copy, so when you want to watch it, it could be easily downloaded from your computer, trustlessly through the multihash, instead of going all the way to your ISP, then to their ISP and 18 hops later, reach google, for the long trip back.

What is ENS?

The Ethereum Naming System (ENS) is a Smart Contract and eco-system built on top of the Ethereum blockchain, which allows anyone to trustlessly purchase a name, which can be thought of a bit like a Domain Name.

Once you have an ENS name, no one can take it away or censor you, and it can be used to trustless associate any information about you that you wish.

How does meeseeks.app work?

The Meeseeks App website was written as a project for the Status #cryptolife hackathon and came in first for the "Infrastructure Track".

It aims to simplify using IPFS for websites, but also improves the security and safety of using IPFS by providing each site with its own domain name, which allows your site to benefit from the decades of work that internet browsers have included to protect you from cross-origin attacks.

When a user accesses your website, for example pac-txt.meeseeks.app, a static bootstrap HTML page is loaded. Everyone gets the same small bootstrap loader, which then performs all the lookups, downloading and verification in the users browser. The steps the bootstrap performs once loaded in the users browser are:

  1. Check the domain name (location.host); if it is a multihash sub-domain, skip to step 3.
  2. Otherwise, the sub-domain is converted to an ENS name (e.g. pac-txt.eth) and resolved against the ENS Smart Contract on Ethereum.
  3. The Multihash is fetched from IPFS; a multihash may consist of links to additional multihashes if it is a large file.
  4. Each multihash is verified against the data fetched from IPFS, and then stitched together.
  5. The bootstrap page then uses document.open to destory and replace itself with the content downloaded from IPFS.
